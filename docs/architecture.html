terraform-aws-network-firewall

AWS Network Firewall with stateful/stateless rules, Suricata IDS/IPS, domain filtering, and centralized inspection VPC pattern
Terraform ≥ 1.5.0 AWS Provider ≥ 5.20.0 Suricata IDS/IPS Strict Rule Order

Executive Summary

This module deploys AWS Network Firewall with a comprehensive firewall policy supporting stateless L3/L4 packet filtering, Suricata-compatible IDS/IPS rule groups for deep packet inspection, domain-based filtering (HTTP Host header and TLS SNI inspection), and 5-tuple stateful rules. It implements the centralized inspection VPC pattern where all inter-VPC and egress traffic is routed through firewall endpoints deployed across multiple Availability Zones. Logging supports both CloudWatch Logs and S3 destinations with ALERT and FLOW log types. S3 log buckets are automatically provisioned with versioning, KMS encryption, lifecycle policies (Standard IA at 30 days, Glacier at 90 days, expiry at 365 days), and public access blocking. The stateful engine uses STRICT_ORDER for predictable rule evaluation.

Component Breakdown

Network Firewall

  • Multi-AZ firewall endpoints via subnet_mapping
  • Deletion protection (firewall, policy, subnet changes)
  • Customer-managed KMS or AWS-owned encryption
  • Firewall policy with strict rule ordering

Stateless Rule Groups

  • L3/L4 packet filtering (protocols, CIDRs, ports)
  • TCP flag matching
  • Priority-based rule evaluation
  • Actions: aws:pass, aws:drop, aws:forward_to_sfe
  • Default action for unmatched + fragmented packets

Stateful Rule Groups

  • Suricata IDS/IPS: full Suricata-compatible rule engine
  • Rule variables (HOME_NET, EXTERNAL_NET)
  • Domain filtering: DENYLIST on HTTP_HOST and TLS_SNI
  • 5-tuple rules for traditional stateful inspection
  • Strict order engine for predictable evaluation

Logging & Storage

  • ALERT and FLOW log types
  • CloudWatch Logs or S3 destinations
  • S3: versioning, SSE (KMS/AES256), lifecycle policies
  • Standard IA at 30d, Glacier at 90d, expire at 365d
  • Public access blocking on all S3 log buckets

Data Flow

  1. Traffic enters through Internet Gateway to public subnets (NAT Gateways)
  2. Route tables direct traffic to Network Firewall endpoints in firewall subnets
  3. Stateless rules process packets first (L3/L4, drop ICMP, forward to stateful engine)
  4. Stateful engine evaluates in strict order: Suricata IDS/IPS, domain filtering, 5-tuple
  5. Suricata rules inspect payload for threat signatures (SSH brute force, C2 channels)
  6. Domain filtering blocks blacklisted domains via HTTP Host and TLS SNI headers
  7. Passed traffic routes through TGW attachment subnets to Transit Gateway
  8. Transit Gateway distributes traffic to spoke VPCs
  9. ALERT logs capture dropped/matched threats; FLOW logs capture all traffic metadata

Industry Adaptation

Enterprise / Hub-Spoke

  • Centralized inspection VPC with Transit Gateway
  • All inter-VPC traffic inspected
  • Suricata rules for threat detection
  • ALERT + FLOW logs to S3 for SIEM integration

Financial Services / PCI

  • Customer-managed KMS encryption
  • Domain filtering for data exfiltration prevention
  • S3 logs with Glacier archival for compliance
  • Deletion protection enabled

Government / FedRAMP

  • IDS/IPS with custom Suricata signatures
  • Strict rule ordering for predictable security
  • Multi-AZ for high availability requirement
  • CloudWatch + S3 dual logging

Production Readiness Checklist

  • Deploy firewall endpoints in at least 2 AZs
  • Enable deletion_protection = true
  • Use CUSTOMER_KMS encryption with a dedicated KMS key
  • Enable both ALERT and FLOW log types
  • Test Suricata rules in alert-only mode before switching to drop
  • Configure domain denylist for known malicious domains
  • Set stateless default action to aws:forward_to_sfe
  • Enable appliance mode on Transit Gateway attachments
  • Review S3 lifecycle policy for log retention compliance
  • Configure route tables to direct all traffic through firewall endpoints
  • Tag all resources with Environment, Security-Zone, ManagedBy
